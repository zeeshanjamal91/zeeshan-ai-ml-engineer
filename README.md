## About

I build AI systems that work outside the lab — in local languages, on constrained infrastructure, for markets that existing tools were not designed for.

My work spans the full ML lifecycle: problem scoping, data pipelines, model development, evaluation, and deployment. I treat software engineering and machine learning as one discipline, not two. When I ship something, I think about latency, failure modes, and the person on the other end of the interface.

I also teach. Translating complex systems into clear, buildable concepts sharpens my own understanding and is something I take seriously.

## Engineering Philosophy

Systems fail at the edges of their assumptions.
Build for the constraints that exist, not the ones you wish for.

### 🔧 Zimma AI — Agentic Local Repair Service Booking

| | |
|---|---|
| **Problem** | Repair and home-service workers in Pakistan operate through informal referrals. There is no scalable discovery or booking layer, and no system that works in Urdu or regional languages. |
| **Solution** | Agentic AI system that handles end-to-end booking in natural language, local languages included. Users describe their problem conversationally; the system classifies the job, finds available workers, and confirms bookings without a human dispatcher. |
| **Key Engineering Challenges** | Intent extraction from informal Urdu/Roman-Urdu input · Worker availability state management in an async pipeline · Graceful fallback when the agent hits ambiguity |
| **Stack** | Python · LangChain · Gemini API · FastAPI · PostgreSQL |
| **Built at** | AI Seekho × Google Antigravity Hackathon |

### 🗺️ UNMAPPED — Informal Economy Skill Recognition

| | |
|---|---|
| **Problem** | Hundreds of millions of workers in LMICs hold verifiable skills with no formal credential. They are invisible to digital labor markets. |
| **Solution** | Offline-first skill recognition system using WebLLM/Phi-3 for on-device inference, mapping informal work descriptions to standardized frameworks (ESCO, O*NET). Multi-tier confidence scoring. Pre-cached static datasets for zero-connectivity environments. |
| **Key Engineering Challenges** | Running inference on-device with < 4 GB RAM · Cross-framework skill mapping with confidence calibration · Offline-first sync architecture |
| **Stack** | Python · WebLLM · Phi-3 · ESCO/O*NET APIs · IndexedDB |
| **Context** | World Bank & MIT innovation context |
